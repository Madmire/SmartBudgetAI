Rapport Complet du Projet SmartBudget AI

Résumé Exécutif

SmartBudget Al est une application web intelligente conçue pour optimiser la gestion des finances personnelles, avec une attention particulière portée aux besoins des étudiants. Le projet vise à transformer la manière dont les utilisateurs interagissent avec leurs budgets en intégrant l'intelligence artificielle et une interface utilisateur moderne. Au cœur de sa proposition de valeur, SmartBudget Al offre un suivi automatisé des revenus et des dépenses, des conseils financiers personnalisés et proactifs, ainsi que des outils de planification d'objectifs d'épargne.

Sur le plan technique, le système repose sur une architecture robuste. Il intègre un agent IA développé en Python, utilisant le framework Langchain et le modèle Gemini 2.0 Flash de Google. Le backend est assuré par FastAPI et PHP, tandis que le frontend est construit avec HTML, CSS et JavaScript, garantissant une expérience utilisateur fluide. Les données sont stockées de manière persistante dans une base de données MySQL. Parmi ses fonctionnalités clés, SmartBudget Al propose la prédiction des dépenses futures, la classification automatique des transactions, un système de définition d'objectifs avec prévisions budgétaires, et des alertes automatiques par e-mail en cas de dépassement de budget. L'application est conçue pour apprendre continuellement du comportement de l'utilisateur, offrant ainsi un accompagnement financier adaptatif et intelligent.

1. Introduction à SmartBudget AI

1.1 Vision et Objectif du Projet

SmartBudget Al est un projet ambitieux qui aspire à redéfinir la gestion des finances personnelles en fusionnant l'intelligence artificielle de pointe avec une expérience utilisateur (UX) moderne. L'objectif fondamental est de permettre aux individus de "gérer leurs finances comme un pro".1 La vision directrice du projet est éloquemment résumée par son slogan : "Un budget maîtrisé, c'est un futur assuré".1

L'essence du projet réside dans sa capacité à fournir un assistant intelligent capable d'apprendre des habitudes de consommation de l'utilisateur. Cette capacité d'apprentissage adaptative permet à SmartBudget Al d'offrir des conseils proactifs, des alertes opportunes et des recommandations intelligentes, aidant ainsi les utilisateurs à prendre des décisions de dépenses plus éclairées et à atteindre leurs objectifs financiers avec efficacité. La conception du système va au-delà du simple suivi transactionnel; elle vise à influencer positivement le comportement financier de l'utilisateur.

1.2 Public Cible et Proposition de Valeur Fondamentale

L'application SmartBudget Al est spécifiquement "optimisée pour les étudiants".1 Ce ciblage répond à des défis financiers uniques auxquels les étudiants sont souvent confrontés, tels que la gestion de revenus fluctuants, le suivi de dépenses variées (logement, alimentation, transport, loisirs) et la planification d'objectifs d'épargne à court et à long terme.

La proposition de valeur fondamentale de SmartBudget Al réside dans sa capacité à automatiser les tâches financières répétitives, à fournir des aperçus personnalisés et exploitables basés sur les habitudes de dépenses individuelles, et à favoriser de meilleures habitudes de consommation grâce à une personnalisation pilotée par l'IA. Cette universalité inhérente suggère un potentiel considérable pour l'expansion du marché au-delà de la population étudiante.

3. Architecture Technique et Pile Technologique

3.1 Architecture Générale du Système

SmartBudget Al fonctionne comme une application web robuste, caractérisée par une séparation claire des responsabilités entre ses composants principaux : un Frontend, une couche d'API Backend, un Agent IA et une Base de Données.1

  • Le Frontend constitue l'interface utilisateur (HTML, CSS, JS).
  • Le Backend, implémenté avec FastAPI et PHP, gère la logique métier.
  • La couche de l'Agent IA, développée en Python avec Langchain et Gemini, représente le cœur intelligent.
  • La Base de Données, alimentée par MySQL, assure le stockage persistant.
  • Un Service d'E-mail intégré utilisant SMTP pour les notifications.

3.2 Ventilation de la Pile Technologique

Composant Technologies Utilisées Rôle/Objectif Principal
Frontend HTML, CSS, Javascript Construction de l'interface utilisateur interactive et réactive.
IA Python, GEMINI Flash 2.0, Langchain Développement de l'agent IA, génération de conseils financiers.
Backend PHP, FastAPI (Python) Gestion de la logique métier (PHP), API pour l'IA (FastAPI).
Base de données MySQL Stockage persistant des données utilisateur, transactions, budgets.

Le choix de "GEMINI Flash 2.0" indique une priorité pour la réactivité et l'efficacité opérationnelle, ce qui est crucial pour une application de finances personnelles où des réponses rapides sont attendues.

4. L'Agent IA de SmartBudget AI : Conception et Capacités

4.1 Composants Clés de l'Agent IA

  • Intégration de Langchain : Sert de couche d'orchestration pour que le modèle de langage interagisse avec des outils externes.
  • Modèle de Langage (LLM) : Utilise ChatGoogleGenerativeAI avec le modèle "gemini-2.0-flash", choisi pour son équilibre entre performance et coût.
  • Type d'Agent : ZERO_SHOT_REACT_DESCRIPTION implémente le paradigme "Reasoning and Acting" (ReAct), permettant à l'agent de décider dynamiquement quel outil utiliser.

4.2 Outils Détaillés de l'Agent et Leurs Opérations

L'agent IA dispose d'une suite d'outils spécialisés pour effectuer des opérations financières. Voici un résumé :

Nom de l'Outil Objectif Interaction Clé
analyze_expenses Analyse les dépenses sur une période donnée. Requête SQL sur la table transactions.
setBudget Définit ou met à jour un budget prévisionnel. INSERT/UPDATE sur la table budgets.
estimerEconomies Estime les économies potentielles. Requête sur les transactions pour évaluer les dépenses.
ajouterTransaction Ajoute une nouvelle transaction (dépense/revenu). INSERT dans la table transactions.
conseillerBudget Analyse les dépenses et génère des conseils financiers personnalisés. Requête sur 3 mois de dépenses, invocation du LLM.
analyseFinanciereMensuelle Fournit un aperçu financier mensuel avec un niveau d'alerte. Calcul du solde basé sur les transactions actuelles et récurrentes.
verifierDepassementBudget Vérifie les dépassements de budget et envoie des alertes par e-mail. Appelé par un planificateur, envoi d'e-mail via mail_send.

5. Détails d'Implémentation du Système

5.1 Backend FastAPI et Point d'Accès API (/ask-agent)

Le point d'interaction principal pour l'IA est un backend FastAPI. Il a été choisi pour ses hautes performances et ses capacités asynchrones. Un point d'accès POST /ask-agent sert de passerelle centrale pour les requêtes de l'utilisateur vers l'agent IA.

Note de sécurité : La configuration CORS allow_origins=["*"] et le stockage des clés API dans un fichier .env sont acceptables pour le développement mais représentent des vulnérabilités de sécurité en production. Des mesures robustes (origines restreintes, gestionnaires de secrets) sont impératives pour un déploiement.

5.2 Tâches d'Arrière-Plan Automatisées

Une tâche planifiée avec BackgroundScheduler s'exécute toutes les 5 minutes pour invoquer l'outil verifierDepassementBudget. Cela assure une surveillance proactive des budgets sans bloquer l'application principale, bien qu'avec une latence maximale de 5 minutes pour les alertes.

5.3 Service de Notification par E-mail (Intégration SMTP Gmail)

Le module mail_send.py utilise la bibliothèque smtplib pour envoyer des e-mails via le serveur SMTP de Gmail. L'authentification est actuellement codée en dur, ce qui constitue une vulnérabilité de sécurité et un point de défaillance unique. Pour la production, il est fortement recommandé d'utiliser des variables d'environnement sécurisées, OAuth 2.0, ou un service d'e-mail transactionnel dédié (ex: SendGrid, Mailgun).

6. Guide d'Installation et de Déploiement

6.1 Clonage du Dépôt

La première étape consiste à cloner le dépôt depuis GitHub :

git clone https://github.com/sanogomamadou/smartbudget_ai.git

6.2 Configuration de l'Environnement

Les clés API pour Gemini et SMTP doivent être configurées dans le fichier .env. Il est crucial de ne jamais commiter ce fichier dans un système de contrôle de version.

6.3 Procédures de Lancement

Les instructions s'appuient sur XAMPP pour le développement local, ce qui est pratique mais non recommandé pour la production.

  1. Backend (FastAPI) :
    • Placer le répertoire smartbudget_ai_agent.
    • Installer les dépendances Python : pip install -r requirements.txt.
    • Lancer le serveur : python main.py.
  2. Frontend (PHP) :
    • Placer les fichiers PHP dans le répertoire xampp/htdocs.
    • Démarrer les services Apache et MySQL depuis le panneau de contrôle XAMPP.

6.4 Accès à l'Application

Une fois les services lancés, l'application est accessible à l'adresse :
http://localhost/smartbudget_ai/pages/sign-up.php

7. Directives de Contribution & Conclusion

SmartBudget Al représente une solution robuste et intelligente pour la gestion des finances personnelles. Son architecture modulaire et ses fonctionnalités proactives basées sur l'IA offrent une base solide pour un développement futur.

Directions Potentielles pour l'Avenir

  • Expansion des Capacités d'IA : Intégrer des conseils d'investissement, des stratégies de gestion de la dette, etc.
  • Intégrations de Services Externes : Se connecter à des API bancaires pour l'importation automatique de transactions (avec une sécurité renforcée).
  • Développement d'Applications Mobiles : Créer des applications natives iOS/Android pour une meilleure accessibilité.
  • Optimisation du Déploiement en Production : Mettre en œuvre une stratégie de déploiement robuste avec conteneurisation (Docker, Kubernetes) et services cloud.
  • Mesures de Sécurité Renforcées : Affiner la gestion des clés API, l'accès à la base de données et les configurations CORS pour un environnement de production.
  • Expansion du Segment Utilisateur : Adapter l'application à d'autres publics comme les jeunes professionnels ou les petites entreprises.
